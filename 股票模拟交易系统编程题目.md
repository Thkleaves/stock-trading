## 二、功能要求

### 1. 前端(Vue 3 + TypeScript,推荐 Vite)

- **登录 / 注册**:本地模拟即可,无需数据库,内存存储
- **行情看板**:
  - 展示至少 3 支股票(代码、名称、最新价、涨跌幅)
  - 价格每秒随机小幅波动(模拟实时行情)
- **交易面板**:
  - 选择股票、输入价格、数量,提交 **买入** 或 **卖出** 委托
  - 显示当前用户的 **委托列表**(未成交 / 已成交)和 **持仓列表**
- **成交记录**:显示最近的成交记录(股票、价格、数量、时间)

### 2. 后端(Node.js + Express / Fastify)

- 提供静态页面入口及 RESTful API
- **内存撮合引擎**:
  - 支持 **限价单**(Limit Order)
  - 使用 **价格优先、时间优先** 规则匹配买卖盘
  - 撮合成功后更新用户持仓和资金(初始资金统一设为 100 万虚拟币)
- 用户数据、委托薄、成交记录均在内存中维护(重启丢失,不要求持久化)

### 3. 实时通信(必做)

- 使用 **WebSocket** 或 **Server-Sent Events (SSE)** 向前端推送:
  - 行情变动
  - 用户委托 / 持仓 / 成交更新
